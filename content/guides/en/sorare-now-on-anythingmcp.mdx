### Why this is a big deal

Sorare's API has two quirks that make it painful to drive from a generic GraphQL client:

1. **Custom auth.** No API keys, no OAuth2. Sign-in is a bcrypt handshake: fetch a per-account salt, hash the password locally, POST the hash through a `signIn` mutation, then echo a JWT-AUD header on every call. The new **`LOGIN_TOKEN`** AuthType in AnythingMCP describes the whole flow declaratively in JSON ‚Äî your password never leaves your server, only the bcrypt hash does, and the JWT cache + auto-refresh are handled for you.
2. **Introspection is disabled on production.** Agents can't discover the schema via `__schema` or `__type`. AnythingMCP's auto-injected `sorare_graphql_schema` tool proxies the SDL through the MCP server and returns task-sized slices (`type:"X"`, `search:"foo"`, `full:true`) so even the largest schemas fit in an agent's context window.

The result: every dedicated tool just works, and when you need something custom your agent can still compose any GraphQL query ‚Äî without needing to know how Sorare's auth works.

### Try it now

- **Hosted (zero install):** sign in on [`cloud.anythingmcp.com`](https://cloud.anythingmcp.com), click **Connectors ‚Üí Sorare ‚Üí Install**, paste your Sorare email + password, mint an MCP API key, point Claude / ChatGPT / Copilot at `https://cloud.anythingmcp.com/mcp`.
- **Self-hosted (full data residency):** `git clone https://github.com/HelpCode-ai/anythingmcp.git && cd anythingmcp && docker compose up -d` ‚Äî same UI on `http://localhost:3000`.

### Notes

- For headless / agent use we recommend a **dedicated read-only Sorare account with 2FA disabled** ‚Äî Sorare's `signIn` rejects requests without a fresh OTP when 2FA is on.
- Sorare expresses fiat amounts in **cents, not currency units**. Divide `eurCents` / `usdCents` / `gbpCents` by 100 ‚Äî Claude / ChatGPT / Copilot read this from the adapter instructions and handle it correctly automatically.
